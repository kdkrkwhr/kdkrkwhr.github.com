블로그 자동 포스팅 작업. workdir는 kdkrkwhr Jekyll 블로그 repo다.

## 절차

1. `.hermes/blog-topics.json` 읽기
2. `status`가 `pending`인 첫 토픽 1개 선택 (posts_per_run 만큼, 기본 1)
3. 해당 토픽 id에 `.hermes/blog-prompt-<id>.txt` 또는 시리즈용 `.hermes/blog-prompt-*.txt`가 있으면 **그 파일을 우선** 따른다 (상세 섹션·소스·날짜 지정)
4. 없으면 `.hermes/blog-style-technical.md` (/technical 프리셋)와 `_posts/` 기존 글 1~2개 참고
5. settings.forbidden 항목(회사·내부 정보) 절대 포함 금지
6. `_posts/YYYY-MM-DD-<slug>.md` 작성 (slug는 토픽 id 또는 영문 kebab-case)
7. **문법 점검**: YAML front matter·코드 블록(```…```)은 제외하고 한국어 본문만 Nara 맞춤법 검사기로 교정 후 반영
8. 해당 토픽 status를 published로, published_at에 ISO 날짜 추가
9. settings.auto_push가 true면 git add, commit, push origin master
10. pending 토픽이 없으면 "작성할 토픽 없음"만 보고하고 종료

## 출력

- 발행한 글 제목, 파일 경로, GitHub Pages URL (https://kdkrkwhr.github.io)
- 남은 pending 토픽 수
- push 실패 시 에러 내용

## 주의

- 이미지는 이번 실행에서 생성하지 않음
- Jekyll front matter 형식 정확히 지킬 것
- 스케줄: **매일 1회** (Hermes cron `0 8 * * *`, KST 08:00)
